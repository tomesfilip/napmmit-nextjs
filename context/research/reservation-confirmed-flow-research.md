### Current State Analysis

### Goal

Napmmit should send a reservation confirmed email and display a reservation confirmed page simultaneously (first show the page and as a back process send the email). Right after the payment for the reservation has been confirmed.

Information to be displayed in the page and in the mail:
  - date of the reservation
  - status
  - number of beds reserved
  - price (and the whole price calculation)
  - contact information on cottage
  - user's contact information provided (his mail/phone number)
  - display a button/link to redirect user to dashboard with reservations (logged user)
  - pdf to download with the same information

A complete implementation plan with:

1. Files to create (with code examples)
2. Files to modify (with specific changes)
3. Testing checklist
4. Implementation order
5. Include mermaid diagrams to explain the flow
